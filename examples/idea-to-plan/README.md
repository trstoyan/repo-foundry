# Example: Idea To Plan

This example shows how a raw idea becomes an agent-operable plan before
implementation starts.

The example is intentionally small and generic. It demonstrates the structure,
not a recommended product.

## Idea

Build a local-first habit tracker for solo builders who want lightweight daily
momentum tracking without accounts or cloud sync.

## Files

- `docs/planning/idea-brief.md`
- `docs/planning/project-plan.md`
- `docs/planning/risk-profile.md`
- `docs/planning/slice-template.md`
- `docs/planning/agent-context.md`

## What This Demonstrates

- An idea is clarified before implementation.
- Non-goals are explicit.
- Risk is classified early.
- The first implementation slice is small and testable.
- Agent context is compact enough for a stateless future agent.
