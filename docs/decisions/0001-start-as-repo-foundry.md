# Decision 0001: Start Repo Foundry as a Living Framework

Date: 2026-05-08
Status: accepted

## Context

We want reusable, agent-ready repository templates based on lessons learned
while working across many active projects. Those lessons are still being
discovered, so a fixed boilerplate would become stale quickly.

## Decision

Create Repo Foundry as a living framework with:

- a small base template;
- a discovery-note process;
- decision records for durable conventions;
- validation scripts that grow as conventions stabilize.

## Consequences

- The first version is intentionally lightweight.
- New rules should be tied to observed project evidence.
- The framework can grow into stack-specific profiles without bloating the
  base template.
