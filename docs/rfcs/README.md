# RFCs

This directory contains larger design proposals for Repo Foundry.

RFCs should be written as reusable public patterns. Do not name private source
repositories or local workspace paths. Use archetypes such as "agentic
development platform", "high-risk runtime system", "multi-project workspace",
or "deployment-heavy service".

RFC lifecycle:

- `draft`: useful idea, not yet adopted into templates or validation.
- `candidate`: reviewed and likely to become a profile or template feature.
- `accepted`: adopted as Repo Foundry guidance.
- `superseded`: replaced by a newer RFC or decision.
