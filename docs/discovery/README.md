# Discovery Notes

Use this directory while touring existing projects. Each note should capture
what was actually observed, why it mattered, and whether it looks reusable.

Prefer small notes over long summaries. A good note is concrete enough that a
future reader can turn it into a template improvement or reject it with context.

## Suggested Filename

```text
YYYY-MM-DD-project-name-short-topic.md
```

## Promotion Path

- `raw`: observed once and not yet generalized.
- `candidate`: observed in more than one place or strongly useful.
- `adopted`: promoted into docs, templates, scripts, or validation.
- `rejected`: intentionally not adopted, with a reason.
