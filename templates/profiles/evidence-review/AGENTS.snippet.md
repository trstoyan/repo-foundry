## Evidence-Based Closeout

Claims are not evidence.

Before accepting a meaningful slice, collect neutral evidence from repository
state and command output. Builder notes may be included only as untrusted
context.

Required closeout evidence:

- git status;
- changed files;
- relevant diffs;
- commands run and raw results;
- skipped checks with reasons;
- deterministic guard results;
- review findings when required;
- release decision or closure record.

Do not approve a slice only because the implementing agent says it is complete.
